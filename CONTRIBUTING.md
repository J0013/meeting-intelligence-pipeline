# Contribuir

Gracias por tu interés. Esto se mantiene simple.

## Cómo contribuir

1. Abre un **issue** describiendo el bug o la propuesta antes de ponerte a programar.
2. Haz un **fork** y crea una rama: `git checkout -b fix/lo-que-sea`.
3. Commits pequeños y descriptivos.
4. Abre un **Pull Request** explicando qué cambia y por qué.

## Estilo

- Mantén el estilo del código existente.
- Si hay tests, que pasen antes del PR.
- Sin dependencias nuevas sin justificar.

## Reportar bugs

Incluye: qué esperabas, qué pasó, pasos para reproducir y tu entorno (SO, versión).
